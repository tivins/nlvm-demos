system.net.TcpListener + system.thread.Thread

Multi-thread HTTP server

This page, style.css and script.js are served by a tiny HTTP server written in NL. Each connection is accepted on the main thread and handed off to its own system.thread.Thread, so requests are handled in parallel — not queued behind one another.

Parallelism check

Fires 5 concurrent requests at /api/work, an endpoint that sleeps 600 ms per request. On a single-threaded server this would take ~3s; here it finishes in ~600 ms because every request runs on its own thread.
